## Intake Summary

User provided a comprehensive design brief covering:
- Full product scope: 13 feature domains (Registration, EMR, Appointments, Scheduling, Follow-up, Prescriptions, Billing, Reports, WhatsApp, Multi-clinic, Patient History, Operations)
- Value proposition: "A complete 360-degree view of every patient's healthcare journey"
- Reference aesthetic: "Stripe meets Linear meets Modern Healthcare"
- Color system: Medical Blue (#0A6EFF), Healthcare Teal (#00B8A9), Soft Cyan (#4FD9FF)
- Typography: Inter / Plus Jakarta Sans / Geist
- 8pt spacing system
- Multi-surface: Mobile + Tablet + Desktop + Large Desktop
- WCAG 2.1 AA target
- No external reference portal/website to analyze
- Theme must be customizable per client (maps to existing multi-tenant CSS variable system)

Additional requirement (second invocation): Theme switching — clients can have in-app theme preset switching.

Stakes: **Consumer/Regulated** (PHI-adjacent data, clinic operations, patient safety context)

## Decisions Made

### Visual Identity

| # | Decision | Rationale |
|---|---|---|
| D-01 | Primary font: Inter (body, labels, data); Plus Jakarta Sans (display, headings) | Inter is the gold standard for dense data UIs; Plus Jakarta Sans adds warmth and premium register for hero moments. Matches Geist-style SaaS without being derivative. |
| D-02 | Sidebar width: 240px expanded, 56px collapsed (icon-only) | Standard enterprise SaaS pattern. Collapsed mode maximizes content area on smaller desktops. |
| D-03 | Border radius: sm=4px, md=8px, lg=12px, xl=16px | Tighter than consumer apps; reads as tool/enterprise. Distinct from Bootstrap-era rounded-pill aesthetic. |
| D-04 | No heavy shadows. Use 1px border + subtle background tint for elevation. | Matches Linear/Notion aesthetic. Heavy shadows = old admin template. |
| D-05 | Card background: #FFFFFF on #F8FAFC page background | Near-white page surface lifts cards cleanly without contrast shock. |
| D-06 | Dark mode: full dark (#0B0F1A base, #141928 card) | Healthcare context includes night-shift staff. Dark mode is a first-class surface, not afterthought. |
| D-07 | Appointment status colors: Blue=Scheduled, Green=Confirmed, Purple=Waiting, Grey=Completed, Red=Cancelled, Amber=No-show | Semantic, not arbitrary. Color-blind safe when combined with text label + icon. |
| D-08 | Patient avatar background: Healthcare Teal (#00B8A9) | Differentiates patient avatars from user/doctor avatars (which use the primary blue). |
| D-09 | Theme tokens are CSS custom properties (--color-primary etc.) injected per client | Maps directly to existing @chikitsa360/branding multi-tenant system. Zero new infrastructure. |
| D-10 | In-app theme switcher: Settings > Appearance — presents preset themes (Light, Dark, High Contrast, Client Brand) | Enables client-level customization without requiring redeployment. Stored in user preferences. |

### Information Architecture

| # | Decision | Rationale |
|---|---|---|
| IA-01 | Left sidebar: Dashboard, Appointments, Patients, Doctors, Prescriptions, Billing, Reports, [divider], Settings | Ordered by daily frequency of use. Prescriptions surfaced separately (not buried under Patients) because doctors use it as a primary surface. |
| IA-02 | Multi-clinic switcher in sidebar header (not a separate nav item) | Admin context switch, not a primary workflow. Clinic name + dropdown at top of sidebar. |
| IA-03 | Patient profile is tabbed: Overview, Timeline, EMR, Prescriptions, Billing, Communication | Tabs prevent scroll fatigue on a data-dense screen. Timeline is the default active tab — most frequent task is "what happened last time?" |
| IA-04 | Appointments: three views: Calendar (week default), Agenda (list), Waitlist | Calendar for scheduling; Agenda for reception workflow; Waitlist for walk-ins. Toggle in top-right. |
| IA-05 | Quick Actions: persistent "+" button in top bar spawns a command palette with: New Patient, Book Appointment, New Prescription, New Invoice | Reduces nav depth to 1 click for all primary creation tasks. |
| IA-06 | Global search (Cmd/Ctrl+K) covers: Patients, Appointments, Doctors, Prescriptions, Invoices, Commands | Linear-style command palette. Fuzzy search. Instant results. Keyboard navigable. |
| IA-07 | Reports is a single surface with tabs: Revenue, Appointments, Patients, Doctors, Follow-ups | Avoids deep report navigation. Common reports surfaced immediately. |
| IA-08 | Settings tabs: Clinic Profile, Users & Roles, Appearance (theme), Integrations, Billing Plan | Appearance tab houses theme preset switcher and custom color overrides. |

### Interaction & Behavior

| # | Decision | Rationale |
|---|---|---|
| I-01 | Keyboard shortcuts: Cmd+K (search/command palette), N (new — context-aware), Esc (close), G then D/A/P etc. (go to section) | Linear-inspired. Power users = clinic staff doing 100+ actions/day. |
| I-02 | Optimistic updates on appointment status changes, patient record saves | Reduces perceived latency. Rollback on failure with toast. |
| I-03 | Appointment booking: 3-step flow (Select Patient → Select Doctor + Time → Confirm) | Research-backed: 3 steps max for booking reduces abandonment. |
| I-04 | Patient registration: smart defaults (city from clinic, date format from locale) | Healthcare forms are abandonment-prone. Smart defaults reduce keystrokes by ~40%. |
| I-05 | All tables: column-sortable, filterable, bulk-selectable, exportable (CSV/PDF) | Enterprise requirement. Reception and admin users work from tables heavily. |
| I-06 | Skeleton loaders everywhere (no spinners blocking layout) | Spinners block reading. Skeletons maintain layout rhythm. |
| I-07 | Toasts: success (3s auto-dismiss), error (persistent until dismissed), info (5s) | Errors must be seen. Success is a brief acknowledgment. |
| I-08 | Drag-and-drop appointment reschedule on calendar (week view, desktop only) | Mobile uses long-press + time picker instead (drag-to-reschedule on mobile is UX anti-pattern). |

### Theme / Multi-tenancy

| # | Decision | Rationale |
|---|---|---|
| T-01 | Default theme: Chikitsa360 light (Medical Blue + Teal + Cyan on white) | The primary brand expression. |
| T-02 | Built-in presets: Light, Dark, High Contrast | Covers accessibility need (High Contrast) without client customization required. |
| T-03 | Client override: Settings > Appearance allows custom primary + secondary color inputs (hex, with live preview) | White-label clients can align to their brand without a new deployment. |
| T-04 | Theme stored in user/device preference (localStorage + server user_preferences) | Persists across sessions. Server-side for multi-device sync. |
| T-05 | CSS custom property architecture: all tokens as --chikitsa-* vars; theme JSON maps 1:1 to these vars | Direct extension of existing branding system in packages/branding/. |

### Mobile

| # | Decision | Rationale |
|---|---|---|
| M-01 | Mobile sidebar becomes a bottom tab bar (5 tabs: Dashboard, Appointments, Patients, + Quick Action FAB, More) | Bottom tabs are thumb-reachable. 5-item limit avoids nav overload. |
| M-02 | Quick Action: Floating Action Button (center of bottom bar) spawns a modal action sheet | Most mobile actions are booking-related. FAB makes it 1-tap reachable. |
| M-03 | Patient profile on mobile: vertical scroll with sticky tab navigation | Tabs above scroll content, sticking at top on scroll. |
| M-04 | Doctor app (tablet focus): EMR/consultation screen is a 2-column split view on tablet | Left: patient summary; Right: current consultation note + prescription builder. iPad landscape optimized. |

### Healthcare Compliance (UX Layer)

| # | Decision | Rationale |
|---|---|---|
| HC-01 | All patient data displays masked by default on Dashboard (names shown; DOB/phone require click-to-reveal) | Privacy-by-default. Protects against shoulder-surfing in clinic settings. |
| HC-02 | Destructive actions (delete patient, cancel appointment) require explicit confirmation dialog with action label (not just "OK") | Error prevention. "Delete Anjali Sharma's record" is safer than "Are you sure?" |
| HC-03 | Session timeout warning: 5-minute idle warning toast before 15-minute auto-logout | PHI protection. Warning prevents surprise logouts mid-consultation. |
| HC-04 | Prescription form: drug name autocomplete from standard formulary, dosage validation, allergy conflict warning | Patient safety. Reduces prescription error. |
| HC-05 | Audit trail: all record edits show "Last updated by {user} at {time}" | Accountability. Required for clinic compliance. |
